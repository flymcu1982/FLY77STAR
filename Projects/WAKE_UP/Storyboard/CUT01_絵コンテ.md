# CUT01 絵コンテ（映画レベル） — WAKE UP（SE77NTH.）

Story Bible(2026-07-10改訂)確定を受けて作成。[[../../../Obsidian_Vault/06_Projects/WAKE_UP/Overview.md|Story Bible]]の内容を基準に、CUT01(0:00〜0:08、渋谷)を実写映画のカット割りとして再構成したもの。生成プロンプト本体は `Storyboard/CUT01.md` を参照。本ファイルはその画コンテ／ショットデザイン部分を詳細化する補助ドキュメント。

## 基準情報

- 尺: 0:00〜0:08（8秒）／BPM 95.7（1拍 ≈ 0.627秒、8秒で約12.8拍）
- 出演: MIU（単独）
- 場所: 渋谷スクランブル交差点付近、夜
- 感情アーク: 雑踏の中の静止 → 気づき（スマホを見る）→ 視線が画面外へ抜ける（CUT02=AYA到着への布石）
- 演出方針: CUT01.mdの「俯瞰ワイド→降下→ミディアム」を5パネルに分解し、それぞれにショットサイズ・カメラワーク・構図・拍タイミングを割り当てる

## Panel 1 — 交差点全体（俯瞰）

- ショットサイズ／レンズ: ELS、広角相当(16-24mm感)
- カメラワーク: 完全俯瞰、固定に近い低速ドローン視点
- 構図:
```
┌────────────────────────────┐
│  ↘   ↙     ↘   ↙    ↘   ↙   │
│     ↙   ↘      ↙   ↘        │
│   （被写体特定なし。人流のみ）  │
└────────────────────────────┘
```
- 芝居: なし(群衆のみ、MIUはまだ画面内で識別されない)
- 補足: ネオン・電光掲示板の光が路面に反射。この段階では「渋谷の熱量」だけを提示し、主人公の存在は伏せる

## Panel 2 — 降下クレーン

- ショットサイズ／レンズ: ELS→WSへの移行、広角のままクレーンダウン
- カメラワーク: 垂直に近い降下。速度は一定、揺れは最小限(手持ち感を出さない)
- 構図:
```
┌────────────────────────────┐
│   ▓▓          ▓▓            │ ← 降下に伴い手前の人影がぼける
│        ↘  ↙      ↘          │
│   ▓▓        ▓▓        ▓▓    │
└────────────────────────────┘
```
- 芝居: なし
- 補足: 高度が落ちるにつれ、ネオンの光量とノイズ(35mmフィルム粒子)が強まる。速度はBPM95.7の推進感に合わせ、急激な加減速はしない

## Panel 3 — MIU、静止（発見）

- ショットサイズ／レンズ: WS、標準〜やや広角(28-35mm感)
- カメラワーク: 緩やかな水平アーク。群衆の間を縫うようにMIUへ視線を誘導
- 構図:
```
┌────────────────────────────┐
│  →     →      →     →       │
│      →     ●MIU    →        │
│  →      (静止)   →     →     │
└────────────────────────────┘
  ●=MIU（画面中央やや下、唯一の静止点）
```
- 芝居: MIUは雑踏の中で一人立ち止まっている。表情はまだ見えない距離感
- 補足: 「動く群衆の中の静止点」という構図でMIUの存在を際立たせる。ここが本カットの核となるショット

## Panel 4 — スマホを見る

- ショットサイズ／レンズ: MS、標準(35-50mm感)
- カメラワーク: ゆっくりとした押し(ドリーイン)。手ブレは最小限
- 構図:
```
┌────────────────────────────┐
│                              │
│        MIU（バストアップ）    │
│        視線: 手元のスマホ     │
│                              │
└────────────────────────────┘
```
- 芝居: 周囲を軽く見渡した後、スマホを一瞬確認する仕草
- 補足: CUT01.mdの「演技」欄と一致させる。表情はまだ落ち着いたトーン

## Panel 5 — 顔を上げる（次カットへの布石）

- ショットサイズ／レンズ: MS〜MCU
- カメラワーク: ほぼ静止、微細な呼吸程度の揺れのみ
- 構図:
```
┌────────────────────────────┐
│                              │
│        MIU（顔を上げる）      │
│        視線: 画面外(→AYA方向) │
│                              │
└────────────────────────────┘
```
- 芝居: スマホをしまい、顔を上げて画面外(CUT02でAYAが現れる方向)へ視線を送る
- 補足: この視線誘導がCUT02の「人混み奥からAYAが手前へ抜けてくるトラッキング」への編集上のモチベーションになる

## パネル間の繋ぎ（編集意図）

- Panel1→2: 俯瞰からの降下はカット無しでワンムーブとして繋ぐ(生成時はカメラパスの分割が必要な場合あり、CUT01.md 4者レビュー内 📷CUT の指摘と一致)
- Panel2→3: 降下が収まるタイミングでMIUの静止点が浮かび上がる。ここで初めて主人公を認識させる
- Panel3→4→5: 芝居はワンカット内の演技の推移として繋ぐ(カット割りせず、押しの中で芝居を変化させる想定)
- Panel5→CUT02: MIUの視線が画面外へ抜けた瞬間でハードカット。CUT01.mdの「編集指示」(CUT02への繋ぎはMIUの視線誘導)と一致

## 生成上のメモ

- CUT01.mdの動画Promptは6-7秒の単一クレーンショットとして設計されているが、Panel1-2(俯瞰〜降下)とPanel3-5(MIUの芝居)は演出の性質が異なるため、生成モデルの精度次第では2クリップに分割してPalmier側で繋ぐ方が安定する可能性がある。最終判断はPalmier編集時に持ち越し
- Panel3-5はMIUの表情の一貫性が重要になるため、Soul ID固定を特に厳密に(CUT01.mdの画像/動画Promptにある"Soul ID locked for consistency"を各パネルの生成でも維持する)
