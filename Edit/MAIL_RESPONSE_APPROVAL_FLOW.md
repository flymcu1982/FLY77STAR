# Mail Response Approval Flow（メール返信承認フロー）

**Document Version:** 1.0  
**Created:** 2026-07-17  
**Status:** 承認フロー確定・実装準備完了

---

## Overview

メール返信を YU が承認・実行する際のフロー。  
AI社員は「下書き作成」までを担当。送信は YU のみ。

**基本原則:**
- AI は返信の「案」を作成
- YU が「承認」してから「送信」
- 返信内容に関する変更・判断は YU が行う

---

## 返信フロー全体図

```
メール受信
    ↓
AI社員が分類・判定
    ↓
「返信が必要」と判定
    ↓
【AI社員】返信下書き作成
    ↓
【YU】下書き確認・修正
    ↓
修正が必要？
├─ はい → AI社員が修正案を作成
│   ↓
│   【YU】再確認
│
└─ いいえ → 【YU】送信実行
    ↓
【AI社員】送信完了を報告
    ↓
メール ✅ DONE ラベル付与
```

---

## Phase 1: AI社員による下書き作成

### Step 1: 返信が必要かを判定

**返信が「必須」の場合:**
- ✅ 仕事依頼への対応 / 断り連絡
- ✅ コラボ依頼への対応 / 検討中の旨
- ✅ ファンメール（感想メール）への感謝返信
- ✅ 緊急対応が必要なメール（確認・状況報告）

**返信が「不要」の場合:**
- ❌ 自動通知（Netlify / GitHub など）
- ❌ SNS成長通知（YouTubeチャンネル成長など）
- ❌ 迷惑メール / スパム
- ❌ 確認メール（既に確認済み）

---

### Step 2: 返信内容の検討

**確認事項:**
- [ ] 何に対する返信か（質問 / 依頼 / 感謝など）
- [ ] 返信の目的は何か（了承 / 検討中 / 断り など）
- [ ] 返信に含めるべき情報は何か
- [ ] トーン（敬語 / カジュアルなど）

**トーンの選択:**

| メール種別 | 適切なトーン | 例 |
|-----------|------------|-----|
| 企業からの仕事依頼 | 敬語・ビジネス | 「かしこまりました。詳細を確認いただきたく...」 |
| クリエイターからのコラボ | 敬語・親しみやすい | 「ありがとうございます。興味深い提案ですね。」 |
| ファンメール | 敬語・あたたかい | 「温かいお言葉をありがとうございます。」 |

---

### Step 3: 返信下書きの作成

**下書きテンプレート:**

```
【返信下書き】

【元のメール情報】
- 送信者: example@company.jp
- 件名: [FW] CM制作依頼について
- 受信日時: 2026-07-17 14:30
- 重要度: 🟠 HIGH

【返信内容】
件名: Re: CM制作依頼について

本文:
お疲れ様です。

このたびはご依頼いただきありがとうございます。

30秒のCM制作についてのご提案、かしこまりました。
ご提示いただきました予算・納期について、確認させていただきたく
存じます。

詳細につきましては、以下の点についてご教示いただけますでしょうか。

1. CMのターゲット層・放映媒体
2. 制作資料・参考映像の有無
3. 修正回数・納品形式

上記をご確認の上、改めてご連絡さしあげます。

何かご不明な点ございましたら、
いつでもお気軽にお問い合わせください。

よろしくお願いいたします。

FLY77STAR. STUDIO
https://flystar77-studio.netlify.app

【AI社員からの注釈】
- 敬語・ビジネストーン使用
- 確認すべき3点を列挙して、丁寧に対応
- 対応意思は明確に伝える
- 連絡先 / Webサイトを署名に記載
```

---

### Step 4: 下書き作成の注意点

**✅ 含めるべき内容:**
- [ ] 相手への感謝・敬意
- [ ] 対応意思（対応する / 検討中 / 検討させていただく など）
- [ ] 次のステップ（確認事項 / 別途連絡予定 など）
- [ ] FLY77STAR の連絡先 / Webサイト
- [ ] 署名（FLY77STAR. STUDIO など）

**❌ 含めてはいけない内容:**
- [ ] 個人的な意見・判断（YU判定が必要）
- [ ] 金額の確定（YU判定が必要）
- [ ] 納期の確定（YU判定が必要）
- [ ] 機密情報・プライベート情報
- [ ] YU が決定していないコミットメント

---

### Step 5: 下書き提出

AI社員が下書きを完成させたら、以下の情報とともに YU に提出：

```
【メール返信案 - 提出】

件名: Re: CM制作依頼について

本文:
[上記の下書き内容]

【AI社員からのコメント】
- トーン: ビジネス敬語
- 確認すべき点を整理して、検討時間を確保する返信
- YU が判定・追加修正した後に送信してください
```

---

## Phase 2: YU による確認・修正・送信

### Step 1: 下書き確認

YU が下書きを受け取ったら、以下を確認：

**チェックリスト:**
- [ ] トーンは適切か
- [ ] 内容に誤りはないか
- [ ] FLY77STAR のブランド価値を損なわないか
- [ ] 機密情報が含まれていないか
- [ ] 相手に失礼がないか

---

### Step 2: 修正・承認の判定

**パターン A: 承認（そのまま送信）**
```
✅ OKです。そのまま送信してください。
```

**パターン B: 修正が必要**
```
修正をお願いします：

1. 1段落目：「検討させていただく」→「検討させていただきたく存じます」に修正
2. 確認事項3）を追加：「概要動画の有無」
3. 署名：「https://flystar77-studio.netlify.app」に統一

修正後、修正案を再度確認してから送信します。
```

**パターン C: 対応不可**
```
こちらはお断りさせていただきたいです。

代わりに、以下の内容で丁寧な断りメールを作成してください：
- 貴社のご提案に感謝
- 現状、難しい旨を明確に
- 将来的な協力の可能性は開いておく
```

---

### Step 3: 修正依頼時の AI社員の対応

YU から修正依頼を受けたら：

1. **修正内容を理解する**
2. **修正案を作成する**
3. **修正理由をコメント付けで返す**

```
【修正案】

本文:
お疲れ様です。

このたびはご依頼いただきありがとうございます。

30秒のCM制作についてのご提案、かしこまりました。
ご提示いただきました予算・納期について、確認させていただきたく存じます。

詳細につきましては、以下の点についてご教示いただけますでしょうか。

1. CMのターゲット層・放映媒体
2. 制作資料・参考映像の有無
3. 概要動画の有無
4. 修正回数・納品形式

上記をご確認の上、改めてご連絡さしあげます。

何かご不明な点ございましたら、
いつでもお気軽にお問い合わせください。

よろしくお願いいたします。

FLY77STAR. STUDIO
https://flystar77-studio.netlify.app

【修正内容】
- 「検討させていただきたく存じます」に修正
- 確認事項4）「概要動画の有無」を追加
```

---

### Step 4: 最終確認・送信

YU が修正案を確認して：

**OK の場合:**
```
✅ 良くできています。これで送信してください。
```

**さらに修正が必要:**
```
1. 「概要動画」→「参考動画」に表現変更してください。

修正後、送信してください。
```

---

### Step 5: メール送信（YU が実行）

修正が完了して最終承認したら、**YU が Gmail から返信を送信**。

**送信時の確認:**
- [ ] To フィールドが正しいか
- [ ] Cc / Bcc が不要か確認
- [ ] 添付ファイルが必要ないか確認
- [ ] 【送信】ボタンをクリック

---

## Phase 3: 送信完了後の対応

### AI社員の役割

1. **送信完了を確認**
   ```
   【返信メール送信完了】
   - 送信先: example@company.jp
   - 件名: Re: CM制作依頼について
   - 送信日時: 2026-07-17 15:30
   ```

2. **メールにラベル更新**
   - ステータス: ⏳ PENDING → ✅ DONE
   - 「返信送信済み」である旨を記録

3. **日報に報告**
   ```
   【返信送信完了】
   - 件数: 1件（高優先度メール）
   - 内容: CM制作依頼への詳細確認返信
   ```

## 返信が難しい場合の対応

### ケース1: 返信内容の判断が必要

**状況:**
AI社員が「返信内容を決定できない」と判定

**対応:**
```
【返信内容判定待ち】

メール: 「マージャンでコラボしませんか」
送信者: collab@game.jp

内容要約:
- ゲーム会社からのコラボ提案
- ゲーム内キャラクター化の打診
- 無料コラボ
- 検討期限: 2026-07-20

→ YU が判定してください：
  1. コラボに応じるか？
  2. 返信内容はどうするか？

YU が判定後、返信下書きを作成します。
```

---

### ケース2: 返信が遅延する場合

**状況:**
受け取ったメールから3日経っても返信していない

**AI社員からのリマインド:**
```
【返信リマインド】

メール: CM制作依頼
送信者: agency@example.jp
受信: 2026-07-17 14:30
状態: ⏳ PENDING（返信待機中）
経過: 3日

推奨: YU から返信をお願いするか、
AI社員が先方に「検討中の旨」を返信してもよいか、
YU に確認してください。
```

---

### ケース3: 返信に含める情報が曖昧

**状況:**
YU が修正指示を出したが、「何を入れるべきか」が不明

**対応:**
```
【修正内容の確認】

修正依頼:
「3）の確認事項を追加してください」

現在の3）:
「修正回数・納品形式」

質問:
確認事項4）として何を追加すればよろしいでしょうか？
→ YU に確認
```
