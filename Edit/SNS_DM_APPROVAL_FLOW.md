# SNS DM 返信承認フロー

**Document Version:** 1.0  
**Created:** 2026-07-18  
**Status:** 承認コマンド仕様確定

---

## 概要

午後 15:00 に Claude Code が生成した SNS DM 返信下書きを、YU が確認・修正・送信の指示を実行するフロー。

---

## YU の承認コマンド仕様

### コマンド形式

```
【SNS-DM-[DM番号]】 【[コマンド]】 [修正内容（オプション）]
```

### 利用可能なコマンド

| コマンド | 機能 | 使用例 | 次のステップ |
|--------|------|--------|-----------|
| **【送信】** | 下書きをそのまま SNS に送信 | 【SNS-DM-001】 【送信】 | 即座に送信実行 |
| **【修正】** | 下書き内容を修正して再提示 | 【SNS-DM-002】 【修正】「料金」→「予算帯」に変更 | 修正内容を反映して再提示 |
| **【確認中】** | 確認中のまま保留 | 【SNS-DM-003】 【確認中】 | 後日判定 |
| **【削除】** | 下書きを破棄・送信しない | 【SNS-DM-004】 【削除】 | 対応終了 |

---

## 各コマンドの詳細

### コマンド 1: 【送信】

**機能:** 下書きをそのまま SNS に送信

**使用シーン:**
- 下書きの内容が完璧で、修正が不要な場合
- 緊急対応が必要な URGENT メール
- 返信内容に満足した場合

**使用例:**
```
【SNS-DM-001】 【送信】
```

**実行後:**
1. Claude Code が DM-001 の下書きを確認
2. 対応プラットフォーム（TikTok / Instagram / X）に実際に DM を送信
3. 送信完了ログを記録
4. YU に「✅ 【SNS-DM-001】送信完了」と報告

**送信完了イメージ:**
```
✅ 【SNS-DM-001】送信完了
プラットフォーム: TikTok
送信先: @xxxx_creator
送信時刻: 15:05
メッセージ: [送信内容]
```

---

### コマンド 2: 【修正】

**機能:** 下書き内容を修正してから再提示

**使用シーン:**
- 表現を修正したい場合
- 特定の単語を別の表現に変えたい場合
- 追加情報を含めたい場合
- トーンを調整したい場合

**使用例（単語置換）:**
```
【SNS-DM-002】 【修正】「料金」→「予算帯」に変更
```

**使用例（追加情報）:**
```
【SNS-DM-003】 【修正】最後に「詳細は fly77star.official@gmail.com までお問い合わせください」を追加
```

**使用例（トーン調整）:**
```
【SNS-DM-004】 【修正】もっと親しみやすい表現に変更
```

**実行後:**
1. Claude Code が修正内容を反映
2. 修正後の下書きを YU に再提示
3. YU が再度確認
4. 【送信】【修正】【確認中】【削除】から改めて指示

**修正後の再提示イメージ:**
```
【修正完了】SNS-DM-002

修正内容: 「料金」→「予算帯」に変更

修正後の下書き:
---
こんにちは！ご依頼・ご質問ありがとうございます。🎬

FLY77STAR. 公式アカウントです。
以下の内容について、詳しくお教えいただけますでしょうか？

✓ プロジェクト概要
✓ 納期・スケジュール
✓ 制作形式（形態・長さなど）
✓ ご予算帯 ← 修正済み

上記をお聞きしたうえで、対応の可否を判断させていただきます。
よろしくお願いいたします！

FLY77STAR. 公式アカウント
---

【修正内容は適切ですか？】
→ 【送信】【修正】【確認中】【削除】から指示してください
```

---

### コマンド 3: 【確認中】

**機能:** 下書きを確認中のまま保留

**使用シーン:**
- 即座に送信するかどうか判定できない場合
- 他のスタッフとの相談が必要な場合
- 後日判定する予定の場合
- メール内容をもっと検討したい場合

**使用例:**
```
【SNS-DM-003】 【確認中】
```

**実行後:**
1. Claude Code がコマンドを記録
2. 下書きは保留状態で保存
3. 翌朝の日報に「確認中: DM-003」として記載
4. 後日 YU が改めて【送信】【修正】【削除】から指示

**保留状態の管理:**
```
【確認中】SNS-DM-003
保留日時: 2026-07-18 15:10
対象: X @creator_collab のコラボ打診
ステータス: 確認中・保留中

翌朝の 18:00 日報に「確認中」として記載
YU が改めて判定するまで送信しない
```

---

### コマンド 4: 【削除】

**機能:** 下書きを破棄・送信しない

**使用シーン:**
- 返信が不要と判断した場合
- スパム・迷惑 DM の場合
- 対応不可と判定した場合
- 送信方針を変更した場合

**使用例:**
```
【SNS-DM-004】 【削除】
```

**実行後:**
1. Claude Code が下書きを削除
2. 対応終了フラグを記録
3. 翌朝の日報に「削除: DM-004」として記載
4. DM-004 は送信されない

**削除のログ:**
```
【削除完了】SNS-DM-004
削除日時: 2026-07-18 15:15
対象: TikTok @spam_user の迷惑メッセージ
理由: スパム判定
送信: なし（対応終了）
```

## 複数 DM への同時指示例

### シナリオ

朝 09:00 で 4 件の DM が抽出され、午後 15:00 に 4 件の下書きが提示された場合。

**YU の指示例:**

```
【SNS-DM-001】 【送信】
→ URGENT 案件。即座に送信。

【SNS-DM-002】 【修正】「ご予算帯」→「予算感」に統一
→ トーン調整のため修正。

【SNS-DM-003】 【確認中】
→ 内容を検討中。明日判定予定。

【SNS-DM-004】 【削除】
→ スパムと判定。対応不要。
```

**Claude Code の実行:**

1. **DM-001:** 即座に TikTok に送信 ✅
2. **DM-002:** 「ご予算帯」を「予算感」に修正 → 再提示
3. **DM-003:** 確認中フラグで保留 ⏳
4. **DM-004:** 削除 🗑️

**YU への報告:**

```
【本日の処理結果】

✅ 【SNS-DM-001】送信完了
  → TikTok に送信しました

⏳ 【SNS-DM-002】修正待機
  → 修正案を再度提示しました。ご確認ください

⏳ 【SNS-DM-003】確認中保留
  → 確認中のまま保留中です。後日の指示をお待ちします

🗑️ 【SNS-DM-004】削除完了
  → スパムとして対応終了しました
```

## DM ステータス遷移図

```
📥 INBOX（受信）
  ↓ 朝 09:00 チェック
👀 REVIEW（分析中）
  ↓ 午後 15:00 下書き生成
📝 DRAFT（下書き完成）
  ↓ YU が承認コマンドを実行
  ├─【送信】
  │  ↓
  │  ✅ SENT（送信完了）
  │
  ├─【修正】
  │  ↓
  │  ⏳ PENDING（修正確認中）
  │     ↓ YU が再度コマンドを実行
  │     ├─【送信】→ ✅ SENT
  │     ├─【修正】→ ⏳ PENDING
  │     ├─【確認中】→ ⏳ PENDING
  │     └─【削除】→ 🗑️ TRASH
  │
  ├─【確認中】
  │  ↓
  │  ⏳ PENDING（保留中）
  │     ↓ 後日 YU が判定
  │     ├─【送信】→ ✅ SENT
  │     ├─【修正】→ ⏳ PENDING
  │     └─【削除】→ 🗑️ TRASH
  │
  └─【削除】
     ↓
     🗑️ TRASH（削除）
```

---

## 重要なルール

### ✅ 実行可能

- YU が【送信】【修正】【確認中】【削除】のコマンドを実行
- YU が DM をブロック・スパム報告
- YU が複数 DM への同時指示

### ❌ 実行不可（Claude Code が勝手には実行しない）

- YU の明示的なコマンドなしでの自動送信
- DM の修正・削除（YU の指示後のみ）
- SNS アカウント操作（パスワード変更など）

## 送信完了後の処理

### 送信ログ記録

```
【送信ログ】
日時: 2026-07-18 15:05
DM ID: SNS-DM-001
プラットフォーム: TikTok
送信先: @xxxx_creator
送信内容: [下書き内容]
ステータス: ✅ 送信完了
```

### 日報への記載

夜 18:00 の日報に以下を含める：

```
【本日の SNS DM 処理結果】

送信完了: 1 件
修正確認中: 1 件
確認中保留: 1 件
削除: 1 件
合計: 4 件

詳細:
- ✅ DM-001（TikTok）送信完了
- ⏳ DM-002（Instagram）修正確認中
- ⏳ DM-003（X）確認中保留
- 🗑️ DM-004（TikTok）削除
```

## トラブルシューティング

### Q: 修正中に追加の修正を入れたい場合は？

**A:** 複数回の【修正】コマンドが可能です。

```
【SNS-DM-002】 【修正】トーンを親しみやすく
→ 修正案を再提示

【SNS-DM-002】 【修正】最後に絵文字を追加
→ 再修正案を再提示

【SNS-DM-002】 【送信】
→ 最終版を送信
```

### Q: 誤って【送信】を実行した場合は？

**A:** 取り消しは不可能です。正確な指示をお願いします。

（SNS 側で DM を削除・編集することはできますが、Claude Code では未送信に戻すことはできません）
