# AI Mail Review Rules（AI メール分析・判定ルール）

**Document Version:** 1.0  
**Created:** 2026-07-17  
**Status:** 分類基準確定・運用中

---

## Overview

AI社員がメールを読み、重要度・カテゴリ・対応要否を判定するための基準書。

**判定の流れ:**
1. メール受信を確認
2. 件名・送信者を見て第一印象を記録
3. 本文を読んで内容を理解
4. 重要度を判定（🔴 / 🟠 / 🟡 / 🟢 / ⚫）
5. カテゴリを決定
6. 対応案を出す
7. YU への報告を作成

---

## Step 1: 件名と送信者の確認

### 件名で初期判定

| パターン | 推定重要度 | アクション |
|---------|----------|---------|
| 「緊急」「エラー」「停止」「制限」「削除」 | 🔴 | すぐに本文を読む |
| 「依頼」「相談」「提案」「対応」 | 🟠 | 詳しく読む |
| 「確認」「通知」「更新」「ニュース」 | 🟡 | 内容を確認 |
| 「営業」「宣伝」「お知らせ」（無関係） | 🟢 | スキャンのみ |
| 「無視」「スパム」「詐欺」の可能性 | ⚫ | 削除候補 |

### 送信者で初期判定

| パターン | 信頼度 | 確認項目 |
|---------|------|--------|
| YouTube / TikTok / Instagram / Google / Netlify / GitHub | ✅ 公式 | 通知内容を確認 |
| example@agency.co.jp / info@company.jp（企業メール） | ✅ 信頼可能 | 本文を詳しく読む |
| creator@gmail.com / artist.name@outlook.com（個人メール） | ⚠️ 要確認 | スパム・詐欺でないか検証 |
| noreply@service.com / no-reply@marketing.jp | ⚠️ 自動通知 | 内容を確認 |
| `example@example.com` / 数字だけのメール | ❌ 疑わしい | スパム・フィッシング可能性あり |

---

## Step 2: 本文の内容理解

### 最初の3行を読む

**第一パラグラフに以下が含まれるか確認:**
- ✅ 自己紹介（企業名・個人名）
- ✅ 連絡目的（「相談」「依頼」「通知」など）
- ✅ 件名の詳細説明

**例1: 仕事依頼**
```
いつもお世話になっております。
△△エージェンシーの田中と申します。
このたび、30秒のCM制作についてご相談したく、ご連絡させていただきました。
```
→ 仕事依頼・高重要度・詳しく読むべき

**例2: スパムメール**
```
こんにちは。
クリックしてください → https://example-spam.jp
```
→ スパム・迷惑メール・すぐ削除

---

### 次に以下を確認

| 項目 | 確認内容 |
|------|---------|
| **何を依頼 / 通知しているのか** | 制作依頼？質問？エラー通知？ |
| **納期・期限はあるか** | 「〇月〇日まで」「お急ぎです」など |
| **予算・金額情報** | 「予算300万」「無料」など |
| **返信が必要か** | 「お返事をお待ちしています」など |
| **添付ファイルはあるか** | 企画書・資料などが添付されていないか |
| **リンク・URLは含まれるか** | 疑わしいリンクが無いか（セキュリティ） |

## Step 3: 重要度の詳細判定基準

### 🔴 緊急（24時間以内に対応が必要）

**以下のいずれかに該当する場合:**

- [ ] YouTube / TikTok / Instagram の「アカウント停止」「制限」「削除」通知
- [ ] サイト / サーバーのダウン通知
- [ ] 支払い不足・請求に関する警告
- [ ] セキュリティ脅威・不正アクセス通知
- [ ] 著作権・法的問題の指摘
- [ ] パートナー / クライアントからの「至急」連絡
- [ ] Google アカウント / Gmail のセキュリティ警告
- [ ] SSL証明書 / ドメインの有効期限切れ警告

**対応:**
→ **すぐに YU に報告** （メール以外で：電話・チャット推奨）

**例:**
```
「YouTube チャンネルが24時間停止されました」
「無料クレジットの残額が30日で切れます」
「不正アクセスが検出されました」
```

---

### 🟠 高（1～3日以内に対応）

**以下のいずれかに該当する場合:**

- [ ] 企業 / クリエイターからの本格的な仕事依頼
- [ ] 有料コラボレーション・パートナーシップ打診
- [ ] 新しい取引先からの初回連絡
- [ ] YouTube / TikTok の広告・収益に関する重要通知
- [ ] API キー / トークンの有効期限切れ予告（30日以上前）
- [ ] SNS 認証 / 2FA に関する新しい要件
- [ ] 取材 / メディア掲載に関する相談
- [ ] 法務・契約に関する事項（危急ではないが重要）

**対応:**
→ **本日中に YU に報告** （メールの日報または別途連絡）

**例:**
```
「CM制作を依頼したいです。予算300万、納期2ヶ月です」
「YouTubeパートナープログラム参加の案内」
「クリエイター取材のお願い」
```

---

### 🟡 中（3～7日以内に確認）

**以下のいずれかに該当する場合:**

- [ ] ファン・視聴者からの一般的な質問
- [ ] SNS の成長通知（「10万フォロワー到達」など）
- [ ] YouTube / TikTok の自動通知（エラーなし）
- [ ] ウェビナー / オンラインイベント情報
- [ ] マーケティング / PR情報
- [ ] ニュースレター / メールマガジン
- [ ] サービスの新機能・アップデート通知
- [ ] コンテスト・応募の案内

**対応:**
→ **週1回の日報に含める** （急ぎではないが確認）

**例:**
```
「SE77NTH.『WAKE UP』すごく好きです！」
「YouTubeチャンネルが100万再生達成しました」
「Netlifyデプロイが完了しました」
```

---

### 🟢 低（確認のみ）

**以下のいずれかに該当する場合:**

- [ ] 無関係な営業メール
- [ ] サービス利用規約 / プライバシーポリシーの更新通知
- [ ] 自動送信メール（確認メール、領収書など）
- [ ] 定期的な確認メール
- [ ] パスワード変更リマインダー
- [ ] ロボット / 自動通知が明らかなメール

**対応:**
→ **報告不要** （分類のみ。YU の確認は不要）

**例:**
```
「パスワードを変更してください」
「サービス利用規約が更新されました」
「ご注文ありがとうございます」
```

---

### ⚫ 迷惑メール / スパム / フィッシング

**以下のいずれかに該当する場合:**

- [ ] 明らかに無関係な営業メール
- [ ] クリックボタン・リンクだけのメール
- [ ] 「アカウント確認」「パスワード更新」を求めるリンク付きメール
- [ ] 怪しいメールアドレスからの身分詐称メール
- [ ] 中国語 / ロシア語など言語不明なメール
- [ ] 「大金が手に入る」「個人情報確認」など詐欺目的の明白なメール

**対応:**
→ **報告後、YU 指示に基づき削除 / スパム報告** （AI が実行しない）

**例:**
```
「クリックしてください → https://fake-bank-login.com」
「アカウントを確認してください」
「あなたは当選しました。クリック」
```

## Step 4: メール分類（6カテゴリ）

### 分類フローチャート

```
メール受信
  ↓
是空ですか？
  ├─ はい → ⚫ スパム
  ├─ 依頼・相談？
  │  ├─ 仕事依頼 → 📋 Work Inquiry
  │  ├─ コラボ依頼 → 🤝 Collaboration
  │  └─ その他相談 → ❓ Other Inquiry
  │
  ├─ プラットフォーム通知？
  │  ├─ YouTube / TikTok / Instagram → 📱 Platform Notification
  │  └─ サービス / システム通知 → ⚙️ Service Notification
  │
  ├─ ファン・一般人から？ → 👥 Fan / Audience
  │
  └─ その他 → 📦 Other
```

### 分類別チェックリスト

#### 📋 Work Inquiry（仕事依頼）

**判定基準:**
- [ ] 「制作依頼」「企画相談」「発注」などのキーワード
- [ ] 企業のメールアドレス（@company.jp など）
- [ ] 具体的な内容・予算・納期が含まれている

**確認すべき情報:**
- [ ] 依頼内容（何を制作するのか）
- [ ] 予算（金額・支払い方法）
- [ ] 納期（〇月〇日まで）
- [ ] 制作ガイド・資料（添付されているか）
- [ ] 決定権者・連絡先（直接担当者か？）

**返信案:**
```
【依頼内容について】
- 予算規模・納期で対応可能かを YU が判定
- 可能な場合：詳細確認メールを送信
- 対応不可の場合：丁寧な断りメール
```

---

#### 🤝 Collaboration Inquiry（コラボレーション依頼）

**判定基準:**
- [ ] 「コラボ」「パートナーシップ」「共同企画」
- [ ] クリエイター / ブランド / イベントからの打診
- [ ] 相互利益を想定した提案

**確認すべき情報:**
- [ ] コラボの内容・形式
- [ ] 報酬・対価（有料か無料か）
- [ ] スケジュール
- [ ] 相手方のフォロワー / 規模

**返信案:**
```
【コラボ判定】
- ブランド価値・時間的余裕を YU が判定
- 可能な場合：詳細条件を確認するメール
- 不可の場合：丁寧な断りメール
```

---

#### 📱 Platform Notification（SNS / 動画プラットフォーム通知）

**判定基準:**
- [ ] YouTube / TikTok / Instagram / X からの自動通知
- [ ] チャンネル成長 / エラー / ポリシー通知

**確認すべき情報:**
- [ ] 通知の種類（成長 / エラー / 制限）
- [ ] 対応が必要か（緊急か？）
- [ ] 詳細情報・リンク

**対応案:**
```
✅ 正常系（成長通知など）
  → 日報に含める

⚠️ 警告系（制限・違反など）
  → 即 YU に報告
```

---

#### ⚙️ Service Notification（サービス / システム通知）

**判定基準:**
- [ ] Netlify / GitHub / Google Cloud / その他のサービスからの自動通知
- [ ] API / トークン / SSL証明書 / デプロイ状況

**確認すべき情報:**
- [ ] サービス名
- [ ] 通知の内容（エラー / 警告 / 情報）
- [ ] 対応期限

**対応案:**
```
✅ 正常系（デプロイ完了など）
  → 報告不要

⚠️ エラー・警告系
  → 日報に含めるか即報告
```

---

#### 👥 Fan / Audience Mail（ファン・視聴者からの連絡）

**判定基準:**
- [ ] 個人メールアドレスからの連絡
- [ ] 「素晴らしい」「好きです」などの感想
- [ ] 質問・リクエスト・応援メール

**確認すべき情報:**
- [ ] 内容の分類（感想 / 質問 / リクエスト など）
- [ ] 返信要望の有無
- [ ] 個人情報・プライバシー情報は含まれていないか

**返信案:**
```
【感想メール】
→ 簡潔な感謝の返信

【質問メール】
→ 回答可能範囲で返信案作成

【リクエスト】
→ 実現可能性を YU に相談後、返信案作成
```

## Step 5: 返信が必要なメールの判定

### 返信の必要性フローチャート

```
メール内容
  ↓
返信が明確に必要か？
  ├─ YES → 返信内容を検討
  │   ├─ 単純な感謝 → 返信案作成
  │   ├─ 情報提供が必要 → 返信案作成
  │   └─ YU判定が必要 → YU に相談後返信案
  │
  └─ NO → 返信不要・報告のみ
```

### 返信が必要な場合の対応

| パターン | 返信内容 | AI の役割 |
|---------|---------|---------|
| 仕事依頼 | 詳細確認メール | 下書き作成 → YU 承認後送信 |
| コラボ依頼 | 検討中の旨・詳細確認 | 下書き作成 → YU 承認後送信 |
| ファン質問 | 回答・感謝 | 下書き作成 → YU 承認後送信 |
| 緊急対応 | 確認・状況報告 | YU が直接対応 |
| 営業メール | 不要（削除） | AI は返信しない |

## Step 6: 対応案の作成

### 対応案テンプレート

```
【メール分析報告】

【基本情報】
- 日時: 2026-07-17 14:30
- 送信者: example@company.jp
- 件名: CM制作依頼について

【判定結果】
- 重要度: 🟠 高
- カテゴリ: 📋 Work Inquiry
- 返信要否: ✅ 返信が必要

【メール要約】
CM制作を依頼したい。
- 内容: 30秒のCM
- 予算: 300万円
- 納期: 2ヶ月
- 決定者: 田中（営業）

【リスク評価】
- 詐欺・スパムの可能性: なし
- 機密情報漏洩リスク: なし
- 実現可能性: YU判定が必要

【推奨アクション】
1. YU が対応可能かを判定
2. 対応する場合：下記の返信案を確認 → 送信
3. 対応不可の場合：丁寧な断り文を YU が作成して送信

【返信案】
（下書きを作成）

【保留事項】
- 相手方の信用調査が必要か
- 予算内で実現可能か
```

## Step 7: メール読み違いを防ぐためのチェックリスト

メール分析を完了する前に、以下を確認してください：

- [ ] 件名と本文の内容が一致しているか
- [ ] 送信者は本当にその企業・個人か（スパム・詐欺でないか）
- [ ] 重要なキーワード（予算・納期・停止・削除など）を見落としていないか
- [ ] 添付ファイル・リンクは安全か
- [ ] 個人情報・秘密情報が含まれていないか
- [ ] 返信が必要なメールを見落としていないか

---

## 判定が困難な場合の対応

**判定に迷った場合:**

```
1. 重要度について迷った
  → より高い方を選ぶ（安全側を選択）
  
2. 分類について迷った
  → 複数の分類に該当する場合、メインの分類を選び、
    サブカテゴリを記載
    例: 📋 Work Inquiry (+ 🤝 Collaboration 要素)
    
3. 返信要否について迷った
  → 「返信が必要な可能性がある」として報告
  → YU が最終判定
    
4. 詐欺・スパムか判断できない
  → 疑わしい場合は ⚫ スパム判定
  → YU に報告して最終判定を仰ぐ
```
