## Overview

Gmail のラベル体系を統一し、メール管理を効率化するためのポリシー。

**ラベルの目的:**
- メールの優先度・内容・進捗を可視化
- 検索の高速化
- YU の確認・対応を支援

---

## ラベル体系（3層構造）

### Layer 1: Priority（優先度ラベル）

メールの重要度を表示。色分けで一目瞭然。

| ラベル | 色 | 意味 | 対応期限 |
|--------|-----|------|---------|
| `🔴 URGENT` | 赤 | 緊急対応が必要 | 24時間以内 |
| `🟠 HIGH` | オレンジ | 重要・高優先度 | 1～3日以内 |
| `🟡 MEDIUM` | 黄 | 通常優先度 | 3～7日以内 |
| `🟢 LOW` | 緑 | 低優先度・参考情報 | 1～2週間 |
| `⚫ SPAM` | 灰色 | 迷惑メール・スパム | 削除対象 |

**使用方法:**
- メール受信時に、重要度に応じてラベルを付与
- URGENT / HIGH の場合は同時にスター（★）も付与
- ラベルは複数付与可（例：HIGH + WORK）

---

### Layer 2: Category（カテゴリラベル）

メールの内容・種類を分類。

| ラベル | 意味 | 例 |
|--------|------|-----|
| `📋 WORK` | 仕事依頼・企画相談 | 制作依頼、企画打診 |
| `🤝 COLLAB` | コラボレーション | クリエイター協力、パートナーシップ |
| `📱 PLATFORM` | SNS / プラットフォーム通知 | YouTube、TikTok、Instagram |
| `⚙️ SYSTEM` | サービス / システム通知 | Netlify、GitHub、Google Cloud |
| `👥 FAN` | ファン・視聴者からの連絡 | 感想、質問、応援 |
| `📧 ADMIN` | 管理・事務 | アカウント確認、パスワード等 |

**使用方法:**
- カテゴリラベルは必ず1つ付与（複数は避ける）
- 複合カテゴリの場合はメインのみ記載

---

### Layer 3: Status（進捗ステータスラベル）

メールの対応状況を追跡。

| ラベル | 意味 | 使用シーン |
|--------|------|----------|
| `📥 INBOX` | 受信・未確認 | 初期状態（全メール） |
| `👀 REVIEW` | 確認中 | AI社員が分析中 |
| `⏳ PENDING` | 保留中・YU判定待ち | YU の判定・承認待ち |
| `📝 DRAFT` | 返信下書き待機中 | 返信案が作成済み・送信待ち |
| `✅ DONE` | 完了・対応済み | 返信送信済み・確認済み |
| `🗂️ ARCHIVE` | アーカイブ | 過去参考情報・検索用に保持 |

**使用方法:**
- ステータスは1つのみ付与
- メールの進捗に応じて更新
- DONE または ARCHIVE で管理完了

## ラベル運用ルール

### ✅ AI社員が実行できる操作

- [ ] ラベル提案：「このメールは HIGH + WORK がいいと思います」
- [ ] 分析報告：「3件が URGENT に分類されました」
- [ ] 検索：「URGENT ラベルのメールを一覧表示」（読み取りのみ）

### ❌ AI社員が実行できない操作

- ❌ ラベルの自動付与（YU 承認後のみ）
- ❌ 新しいラベルの作成（YU のみ）
- ❌ ラベルの削除（YU のみ）
- ❌ ラベルの色変更（YU のみ）

## Gmail フォルダ vs ラベル

Gmail では従来の「フォルダ」ではなく「ラベル」を使用。

**ラベルの利点:**
- 1つのメールに複数ラベル可（フォルダは1つのみ）
- ラベルなしでも「受信トレイ」に残る
- 検索が容易

**運用のコツ:**
- ARCHIVE ラベルが付いたメールは「すべてのメール」から検索可
- 完全に削除する必要がない限り、アーカイブで管理

---

## Gmail スター（★）の使い方

スターは「最も重要」を示す補助機能として使用。

**スター付与ルール:**
- [ ] URGENT ラベルのメール → 必ずスター付与
- [ ] HIGH で返信が必要 → スター付与を検討
- [ ] その他 → 不要

**スター確認:**
- YU が朝確認：「スター付きメール」を一覧表示
- AI社員：スターの個数で優先度を把握

## Gmail フィルター設定（YU が設定）

ラベルの自動付与を設定して、手作業を削減。

### 推奨フィルター設定例

```
1. YouTube 自動ラベル
   条件: From:youtube-noreply@google.com
   アクション: ラベル追加 📱 PLATFORM + 🟡 MEDIUM

2. GitHub 自動ラベル
   条件: From:notifications@github.com
   アクション: ラベル追加 ⚙️ SYSTEM + 🟢 LOW

3. Netlify 自動ラベル
   条件: From:noreply@netlify.com
   アクション: ラベル追加 ⚙️ SYSTEM + 🟢 LOW

4. 迷惑メール 自動ラベル
   条件: From:(spam domain リスト)
   アクション: ラベル追加 ⚫ SPAM + スキップ
```

**設定権限: YU のみ**

```

├─ 新規メールを確認（必要に応じて返信指示）
└─ ラベル提案を承認 / 修正

夜 18:00 - AI社員
├─ 本日のメール総まとめ
├─ ラベル統計を作成
│  （URGENT 件、HIGH 件など）
└─ 日報に含める

就寝前 20:00 - YU
└─ URGENT メールをチェック
```
