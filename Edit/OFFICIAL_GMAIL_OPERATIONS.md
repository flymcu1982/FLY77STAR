**Document Version:** 1.0

## Overview

FLY77STAR 公式Gmail（`fly77star.official@gmail.com`）の運用体制。  
AI社員による「受信・分類・報告」と、YU による「承認・実行」の二層構造。

**運用フェーズ:**
- **Phase 1 (現在):** AI社員=読み取り専用・分類・報告のみ
- **Phase 2 (将来):** AI社員=返信下書き作成・YU承認後送信
- **Phase 3 (次段階):** Gmail API / OAuth 統合（安全ルール別途設定）

---

## 基本ルール

### 🔐 セキュリティ最優先

| Rule | 詳細 |
|------|------|
| **パスワード非共有** | Gmailパスワードは YU のみが管理・使用。AI には渡さない。|
| **ログイン管理** | Gmail ログイン・OAuth 認証・権限付与は YU 本人のみが実行。|
| **アクセス記録** | AI が Gmail を確認した際は、日時・確認者・概要を報告。|
| **情報保護** | 個人情報・案件情報・契約情報は、確認後も外部共有しない。|
| **承認フロー** | 返信・削除・ラベル変更・アーカイブは、すべて YU 承認が必須。|
| **送信禁止** | AI が勝手に送信・返信・自動応答を実行しない。|

## AI社員の役割定義

### Mail Reception Department（受信窓口部門）

**職務:**
- 毎日のメール受信を確認
- 件名・送信者・本文の要点を読む
- 重要度を判定（緊急 / 高 / 中 / 低）
- メール分類ラベルを提案
- YU への報告フォーマットを作成

**使用ツール:**
- Gmail UI（ブラウザまたはモバイル）
- 内部ドキュメント（報告フォーマット）

**できること:**
- ✅ メール受信状況を確認
- ✅ 件名・本文の要点を読み・要約
- ✅ 送信者の背景情報を調べる（オンライン検索など）
- ✅ 重要度・内容を分類
- ✅ 返信案・対応案を作成（下書き）
- ✅ 日報を作成

**できないこと:**
- ❌ パスワードを変更・管理
- ❌ Gmail 設定を変更
- ❌ メールを削除
- ❌ メールをアーカイブ
- ❌ ラベルを作成・変更
- ❌ メールを送信・返信
- ❌ フィルタルールを設定
- ❌ 本当のメール操作（YU承認なし）

## 毎日確認する項目

### 朝（毎日 09:00 前）

1. **未読メール数を確認**
   - 前日からの未読は何件か
   - スターは付いているか

2. **重要度「高」「緊急」のメールをチェック**
   - 仕事依頼・コラボ依頼・緊急通知を優先
   - 件名と本文の冒頭を読む

3. **サービス通知を確認**
   - Netlify / GitHub / Google / その他の自動通知
   - エラーや警告がないか確認

4. **ファン・視聴者からのメールをチェック**
   - SNS連動投稿がある場合、反応メールが来てないか

### 昼（毎日 12:00 前後）

5. **新着メールをスキャン**
   - 午前 09:00 以降のメール新規受信を確認

### 夜（毎日 18:00 以降）

6. **本日のメール総まとめ**
   - 重要なメールをピックアップ
   - 返信が必要なメールをリスト化
   - YU への日報を作成

## メール重要度の判定基準

### 🔴 緊急（24時間以内に対応が必要）

**判定基準:**
- サイトエラー / サーバーダウン通知
- 支払い・納期に関わる警告
- 著作権 / 法的問題の指摘
- クライアント / パートナーからの至急連絡
- YouTube / TikTok アカウント停止 / 制限通知

**対応:**
→ すぐに YU に報告（電話・チャット推奨）

---

### 🟠 高（1～3日以内に対応）

**判定基準:**
- 仕事依頼（製作依頼・企画相談など）
- 本格的なコラボレーション打診
- パートナーシップ・案件に関する重要なメール
- 新規取引先からの初回連絡
- SNS認証 / API キーの有効期限切れ通知

**対応:**
→ 本日中に YU に報告

---

### 🟡 中（3～7日以内に確認）

**判定基準:**
- ファン・視聴者からの一般的な質問
- SNS / YouTube / TikTok の自動通知（エラーなし）
- ウェビナー / イベント情報
- マーケティング情報の受信
- 定期ニュースレター

**対応:**
→ 週1回の定期報告に含める

---

### 🟢 低（確認のみ）

**判定基準:**
- 営業メール / 勧誘メール
- サービス利用規約の更新通知
- パスワード変更リマインダー
- 登録確認メール（既に確認済みの場合）

**対応:**
→ 分類のみ。報告は不要。

---

### ⚫ 迷惑メール / スパム

**判定基準:**
- 無関係な営業
- フィッシングメール
- 明らかな詐欺メール

**対応:**
→ 報告し、YU 指示に基づき削除 / スパム報告

## メール内容の分類

### Category 1: 仕事依頼（Work Inquiry）

**特徴:**
- 「制作依頼」「企画相談」「制作を受け付けていますか」など
- 納期・予算・詳細条件を含むことが多い

**初期対応:**
1. 依頼内容を要約する
2. 予算・納期・作品形式を整理する
3. YU が対応可能かを判定してもらう

**例文:**
```
【仕事依頼】CM制作の相談

送信者: ✉️ example@agency.co.jp
重要度: 🟠 高

概要:
30秒のCM制作を依頼したいと連絡。予算300万・納期2ヶ月。
AI動画生成のポートフォリオがあるかを質問。

対応案:
→ YU が応対を判定
→ 対応する場合: 詳細条件を確認するメール下書き
→ 対応しない場合: 丁寧な断りメール下書き
```

---

### Category 2: コラボレーション依頼（Collaboration Inquiry）

**特徴:**
- 「コラボしませんか」「相互協力」「参加しませんか」
- クリエイター / ブランド / メディアからの打診

**初期対応:**
1. コラボ相手と内容を整理する
2. FLY77STAR のメリット・デメリットを検討
3. YU が判断してもらう

**例文:**
```
【コラボ依頼】音楽フェス出展の打診

送信者: ✉️ festival@event.co.jp
重要度: 🟠 高

概要:
8月の音楽フェスに SE77NTH. の映像出展を依頼。
無料出展・YouTubeで配信予定。

対応案:
→ フェスの規模・視聴者数を調べる
→ YU が判定（ブランド価値・時間的余裕など）
→ 対応案メール下書き
```

---

### Category 3: SNS / 動画プラットフォーム関連通知（Platform Notifications）

**特徴:**
- YouTube / TikTok / Instagram からの自動通知
- チャンネル成長 / コンテンツ削除 / 制限通知

**初期対応:**
1. 通知内容を読む（エラー・警告がないか確認）
2. 対応が必要かを判定
3. 緊急の場合は YU に報告

**例:**
- ✅ YouTube: 100万再生達成通知 → 報告
- ✅ TikTok: フォロワー10万到達 → 報告
- ✅ Instagram: ストーリー機能解放 → 報告
- 🔴 YouTube: コンテンツポリシー違反警告 → 緊急報告

---

### Category 4: サービス / システム通知（Service Notifications）

**特徴:**
- Netlify / GitHub / Google Cloud などの自動通知
- API キー / SSL証明書 / デプロイ状況

**初期対応:**
1. 通知内容を確認
2. エラー・警告・更新があるか把握
3. 必要に応じて YU に報告

**例:**
- ✅ Netlify: デプロイ完了 → 報告不要（正常系）
- 🟠 GitHub: セキュリティアラート → 報告必須
- 🔴 Google Cloud: 無料クレジット残少 → 報告必須

---

### Category 5: ファン・視聴者からの連絡（Fan / Audience Mail）

**特徴:**
- 「素晴らしい作品ですね」「好きです」などの感想
- 質問・リクエスト・コラボ提案など

**初期対応:**
1. 内容を分類（感想 / 質問 / 要望など）
2. 返信が必要かを判定
3. 返信案を作成

**例:**
```
【ファンメール】SE77NTH. への感想

送信者: fan@example.com
重要度: 🟡 中

本文: 「SE77NTH.『WAKE UP』最高でした。
次の作品はいつ公開ですか？」

対応案:
→ 感謝の返信 + 次作品の情報を伝える（YU が承認後送信）
```

---

### Category 6: 営業 / スパム / 迷惑メール（Marketing / Spam）

**特徴:**
- 無関係な営業メール
- 「サービス導入しませんか」系の勧誘
- フィッシング・詐欺メール

**初期対応:**
1. 迷惑メール判定
2. スパム報告 / 削除を YU に提案

**例:**
```
【迷惑メール】
送信者: noreply@spam-service.com
理由: 無関係な営業メール
処置: 削除 / スパム報告を推奨
```

---

### Category 7: 緊急対応が必要なメール（Urgent / Crisis）

**特徴:**
- サイト / サーバーのダウン
- 著作権侵害指摘
- アカウント停止 / 制限
- 支払い / 契約に関わる問題

**初期対応:**
1. **すぐに YU に連絡**（メール通知ではなく、電話・チャット推奨）
2. 詳細を要約
3. 示唆される対応を簡潔に提案

**例:**
```
【緊急】YouTube チャンネル一時停止通知

通知日時: 2026-07-17 14:30
重要度: 🔴 緊急

内容: 「ポリシー違反により、チャンネルが24時間停止」

対応案:
1. YouTubeの異議申し立て制度を確認
2. 違反内容を詳しく確認
3. 必要に応じて申し立て
→ 即座に YU に報告してください
```

### 日報テンプレート

```

【受信サマリー】
- 本日受信: 15件
- うち未読: 8件
- 昨日からの繰越未読: 3件

【重要度別分類】
🔴 緊急: 1件（YouTube アカウント警告）
🟠 高: 3件（仕事依頼2件、コラボ依頼1件）
🟡 中: 6件（ファン質問4件、サービス通知2件）
🟢 低: 5件（営業メール）

【本日のハイライト】

1. 【仕事依頼】CM制作 30秒
   送信者: agency@example.com
   予算: 300万円
   納期: 2ヶ月
   → YU判定待ち

2. 【コラボ依頼】音楽フェス出展
   イベント名: Summer Music Fest 2026
   出展形式: SE77NTH. 映像放映
   集客: 予想5000人
   → YU判定待ち

3. 【緊急】YouTube ポリシー違反警告
   チャンネル: FLY77STAR OFFICIAL
   状態: 24時間停止中
   原因: 不明確（詳細確認が必要）
   → 即報告

【返信が必要なメール】
- ファンメール「次作品の公開時期」 → 返信案あり

【推奨アクション】
1. YouTube ポリシー違反の詳細確認
2. 仕事依頼・コラボ依頼の YU 判定
3. YouTube 異議申し立てメールの準備

【その他特記事項】
- Netlify デプロイ正常
- GitHub セキュリティアラートなし
- Google Cloud 費用正常
```

## YU が承認すべき操作

| 操作 | AI が実行可能？ | 備考 |
|------|----------------|------|
| メール読み取り | ✅ はい | 読み取り専用 |
| 要約・分類 | ✅ はい | 分類提案のみ |
| 返信下書き | ✅ はい（今後） | YU承認後送信 |
| メール送信 | ❌ いいえ | YU承認後 YU が送信 |
| メール削除 | ❌ いいえ | YU承認後 YU が削除 |
| メールアーカイブ | ❌ いいえ | YU承認後 YU が実行 |
| ラベル変更 | ❌ いいえ | YU承認後 YU が実行 |
| フィルタ設定 | ❌ いいえ | YU のみが設定 |
| 自動返信設定 | ❌ いいえ | YU のみが設定 |
| 転送設定 | ❌ いいえ | YU のみが設定 |

## 将来の API / OAuth 連携時のルール

Gmail API や MCP を使って自動化する場合の安全ルール：

### ✅ 自動化してよい操作

- メール読み取り・分類・ラベル付与提案
- 重要メールの自動抽出・通知
- 日報の自動生成

### ❌ 自動化してはいけない操作

- メール送信（YU 承認が絶対必須）
- メール削除（YU 承認が絶対必須）
- 返信の自動送信（下書き作成まで）
- ラベル自動変更（提案のみ）

### 🔐 セキュリティ要件

- API キー / OAuth トークンは環境変数で管理
- トークンをログ・履歴に残さない
- アクセス可能な範囲を最小限に限定
- 読み取り専用スコープを優先
- 定期的にアクセス権限を監査
