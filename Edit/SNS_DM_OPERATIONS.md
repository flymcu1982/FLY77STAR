## 概要

FLY77STAR. 公式 SNS（TikTok、Instagram、X）に到着する DM（ダイレクトメッセージ）を、以下のフローで自動管理：

1. **取得・分類** → Claude Code が毎朝チェック・分類
2. **下書き作成** → Claude Code が返信案を自動生成
3. **YU 確認** → YU が内容を確認・修正
4. **YU 承認** → YU が「送信」コマンドを実行
5. **自動送信** → Claude Code が SNS に送信

## 運用スケジュール

### 毎朝 09:00 — SNS DM Daily Check

**実行:** Claude Code  
**ファイル:** `SNS_DM_DAILY_CHECKLIST.md`

**内容:**
- TikTok、Instagram、X 全プラットフォームの新着 DM 確認
- 各 DM を分類（INQUIRY / COLLAB / FAN / BUSINESS / URGENT / SPAM）
- 重要度判定（URGENT / HIGH / MEDIUM / LOW）
- DM 件数を集計
- 本日の対応対象 DM を抽出

**出力:** チェックシートに記入

---

### 午後 15:00 — SNS DM Draft Preparation

**実行:** Claude Code  
**ファイル:** `SNS_DM_DRAFT_PREPARATION.md`

**内容:**
- 朝のチェックシートから対応対象 DM を確認
- DM タイプ別テンプレートを選択
- 各 DM に対して返信下書きを自動生成
- 「送信可能な状態」で YU に提示

**出力:** 下書きテンプレート（YU 確認待ち）

---

### YU 承認フロー — DM 返信承認

**実行:** YU  
**ファイル:** `SNS_DM_APPROVAL_FLOW.md`

**内容:**
- Claude Code が提示した下書きを確認
- 修正が必要な場合は修正コマンドを実行
- 送信準備完了なら「送信」コマンドを実行

**承認コマンド:**
```
【確認中】— 確認中の状態を維持
【修正】xxx — xxx の部分を修正
【送信】— 実際に SNS に送信
【削除】— 下書きを破棄
```

---

### 送信実行 — SNS 自動送信

**実行:** Claude Code（YU 「送信」承認後）  
**内容:**
- 該当プラットフォーム（TikTok / Instagram / X）に DM 送信
- 送信ログを記録
- 送信完了を YU に報告

**送信対象:**
- 「送信」承認を受けた下書き
- 複数承認がある場合は順序を維持

---

### 夜 18:00 — SNS DM Daily Report

**実行:** Claude Code  
**ファイル:** 共通日報に含める

**内容:**
- 本日の DM 受信件数（プラットフォーム別）
- 分類別集計
- 送信済み DM 件数
- 承認待ち DM 件数
- 特記事項（URGENT の対応など）

**出力:** YU に日報で報告

## 重要ルール

### ✅ Claude Code（AI社員）が実行できる操作

- [ ] DM 自動取得・確認
- [ ] DM の分類・タイプ判定
- [ ] 重要度判定
- [ ] 返信下書き自動生成
- [ ] 下書き案を YU に提示
- [ ] 送信ログ記録
- [ ] 日報作成・報告

### ❌ Claude Code（AI社員）が実行できない操作

- ❌ DM ログインのためのパスワード管理
- ❌ YU 承認なしの実際の DM 送信
- ❌ DM ブロック・削除（YU の指示後のみ）
- ❌ SNS 設定変更・アカウント操作

### 👤 YU のみが実行できる操作

- 👤 SNS にログイン
- 👤 DM を直接読む
- 👤 「送信」「修正」「削除」のコマンド実行
- 👤 DM ブロック・スパム報告
- 👤 SNS 設定変更

## SNS DM 承認コマンド仕様

### YU が Claude Code に出す指示

```markdown
【SNS-DM-[番号]】 【送信】

説明：指定番号の DM 下書きを実際に SNS に送信する
```

```markdown
【SNS-DM-[番号]】 【修正】[修正内容]

説明：下書きを修正。修正内容を記載する
```

```markdown
【SNS-DM-[番号]】 【確認中】

説明：確認中のままキープ。後日判定する
```

```markdown
【SNS-DM-[番号]】 【削除】

説明：下書きを破棄。送信しない
```

**例:**
```
【SNS-DM-001】 【送信】
→ DM-001 を即座に TikTok に送信

【SNS-DM-002】 【修正】「納期」を「期限」に変更
→ DM-002 の下書き内で「納期」を「期限」に修正して、修正内容を提示

【SNS-DM-003】 【確認中】
→ DM-003 は承認待ちのままキープ
```

## DM ステータス管理

| ステータス | 意味 | 次のステップ |
|-----------|------|-----------|
| **📥 INBOX** | 新規受信 | 朝のチェックで分類 |
| **👀 REVIEW** | 分類・分析中 | 午後に下書き準備 |
| **📝 DRAFT** | 返信下書き完成 | YU が「送信」確認待ち |
| **⏳ PENDING** | 修正確認中 | 修正後に「送信」判定 |
| **✅ SENT** | 送信完了 | 日報に記録・終了 |
| **🗑️ TRASH** | 削除済み | 対応終了 |
