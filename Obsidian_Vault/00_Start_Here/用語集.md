# 用語集

専門用語は必ずこの形式(用語名 → 一行説明)で載せます。わからない言葉が出てきたら、まずここを見てください。新しい用語を作ったら、このページに追記するのがルールです。（[[Vault運用ルール]] 参照）

## Vaultの仕組みに関する用語

Founder's Journal
（創業者の想い・考えを残すノート）

Discovery Log
（今日の気づきノート）

Knowledge
（正式採用された会社ルール・ノウハウ）

Daily Studio Report
（毎日AM/PMに社長が確認する30秒日報）

ハブノート
（ある名前を持つものについて、正式な説明を1つだけ載せるノート。検索の着地点になる）

索引
（Vault内の主なハブノートを一覧できる、検索の保険ページ）

Characters
（作品内のキャラクター。実在のスタッフを指すPeopleとは分けて管理する）

MOC
（Map of Content。そのフォルダの目次になっているノートのこと）

Vault
（Obsidianで開く、ノート一式が入ったフォルダ全体のこと）

昇格
（Discovery Logに書いた気づきを、見直した上でKnowledgeなど正式な場所に移すこと）

昇格候補
（異なる日に3回以上使われるなど、Knowledgeに昇格させてよさそうな発見に付けるタグ）

昇格済み
（実際にKnowledgeへ昇格が完了した発見に付けるタグ）

レビュー済み
（Knowledge Reviewで見直したが、昇格はしなかった発見に付けるタグ）

Knowledge Review
（未レビューのDiscovery Logが10件たまったら行う、Knowledgeへの昇格を検討する見直し作業）

収穫の儀式
（Discovery LogのKnowledge Review、Founder's Journalの半年〜年次の振り返り、Knowledgeの年次棚卸しをまとめた呼び方）

Vault Manager
（制作中に決まった内容を、Daily Log・Discovery・Decision Log・Character/Costume/MV Bible・Prompt Libraryへ自動で振り分け保存するツール。実体はGitHub上のscripts/vault_manager.py）

変更履歴
（Vault Managerが自動で行った変更を、日時・操作・対象で記録する追記専用ページ）

Character Bible
（各キャラクターの設定を一元管理する場所。実体は10_Characters/）

Costume Bible
（衣装デザインと、それを採用した理由を一元管理する場所。実体は11_Costume_Bible/。却下案はCostume BibleではなくDecision Logに残す）

MV Bible
（作品ごとに脚本・絵コンテ・CUT・プロンプトを管理する場所。実体は06_Projects/の各Overview.md）

Decision Log
（採用したもの・却下したもの・採用理由を、決定のトピックごとに履歴として残す場所。実体は03_Company/Decisions/）

Daily Log
（今日やったこと・完成したこと・次回やることを自動記録する仕組み。実体はDaily Studio Report内の3つの自動記録セクション）

## ブランド・作品に関する用語

FLY77STAR.
（会社/レーベルの正式表記。読み:「フライスター」— 77は読まない。末尾のドットまで含めて正式。Luka Dončićの背番号77へのオマージュ）

SE77NTH.
（3人組AIガールズグループ。読み:「セブンス」— 77は読まない。詳細: [[10_Characters/SE77NTH|SE77NTH.]]）

Soul ID
（Higgsfield Soul 2.0でキャラクターの顔を固定登録する仕組み。同一キャラは全生成で同じSoul IDを使い、顔ドリフトを防ぐ）

77ルール
（ブランド・グループ名の綴り中に「77」を視覚的に埋め込み、発音はしない命名規則。「なんて読むの?」という検索フックを作る）

era（時代管理）
（同一キャラクターのビジュアル大変更を「時代」として区別し、Soul IDも時代ごとに別登録する運用。例: KAIのMA-1期/US期）

## 制作に関する用語

Production Bible
（映像制作マニュアル。世界観・キャラクター・品質基準をまとめた文書）

CUT
（作品を構成する一つの映像カットの単位。CUT01〜のように番号で管理する。総数は作品ごとに異なる（例: WAKE UPは31、STAR LiGHTは50）。AI社員の [[04_Team/AI_Employees/CUT (AI Employee)|CUT (AI Employee)]] とは別物なので注意）

EDIT_PLAN.md
（CUT_LIST.jsonから自動生成される、CUTごとの尺と編集指示をまとめた表）

edit_project.json
（Palmierに渡す、タイムライン全体の編集用データファイル）

CUT_LIST.json
（各CUTの役割・尺・セリフ有無・素材パスを管理する元データ）

Palmier

Higgsfield / Nano Banana

Seedance / Gemini Omni Flash
（MVや動画、リップシンクを生成するツール）

ElevenLabs
（ナレーションや声を生成するツール）
