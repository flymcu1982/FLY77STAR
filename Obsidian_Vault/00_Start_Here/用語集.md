# 用語集

専門用語は必ずこの形式(用語名 → 一行説明)で載せます。わからない言葉が出てきたら、まずここを見てください。新しい用語を作ったら、このページに追記するのがルールです。（[[Vault運用ルール]] 参照）

## Vaultの仕組みに関する用語

Founder's Journal
（創業者の想い・考えを残すノート）

Discovery Log
（今日の気づきノート）

Knowledge
（正式採用された会社ルール・ノウハウ）

Daily Studio Report
（毎日AM/PMに社長が確認する30秒日報）

ハブノート
（ある名前を持つものについて、正式な説明を1つだけ載せるノート。検索の着地点になる）

索引
（Vault内の主なハブノートを一覧できる、検索の保険ページ）

Characters
（作品内のキャラクター。実在のスタッフを指すPeopleとは分けて管理する）

MOC
（Map of Content。そのフォルダの目次になっているノートのこと）

Vault
（Obsidianで開く、ノート一式が入ったフォルダ全体のこと）

昇格
（Discovery Logに書いた気づきを、見直した上でKnowledgeなど正式な場所に移すこと）

昇格候補
（異なる日に3回以上使われるなど、Knowledgeに昇格させてよさそうな発見に付けるタグ）

昇格済み
（実際にKnowledgeへ昇格が完了した発見に付けるタグ）

レビュー済み
（Knowledge Reviewで見直したが、昇格はしなかった発見に付けるタグ）

Knowledge Review
（未レビューのDiscovery Logが10件たまったら行う、Knowledgeへの昇格を検討する見直し作業）

収穫の儀式
（Discovery LogのKnowledge Review、Founder's Journalの半年〜年次の振り返り、Knowledgeの年次棚卸しをまとめた呼び方）

Vault Manager
（制作中に決まった内容を、Daily Log・Discovery・Decision Log・Character/Costume/MV Bible・Prompt Libraryへ自動で振り分け保存するツール。実体はGitHub上のscripts/vault_manager.py）

変更履歴
（Vault Managerが自動で行った変更を、日時・操作・対象で記録する追記専用ページ）

Character Bible
（各キャラクターの設定を一元管理する場所。実体は10_Characters/）

Costume Bible
（衣装デザインと、それを採用した理由を一元管理する場所。実体は11_Costume_Bible/。却下案はCostume BibleではなくDecision Logに残す）

MV Bible
（作品ごとに脚本・絵コンテ・CUT・プロンプトを管理する場所。実体は06_Projects/の各Overview.md）

Decision Log
（採用したもの・却下したもの・採用理由を、決定のトピックごとに履歴として残す場所。実体は03_Company/Decisions/）

Daily Log
（今日やったこと・完成したこと・次回やることを自動記録する仕組み。実体はDaily Studio Report内の3つの自動記録セクション）

FLY77STAR Universe
（FLY77STARの全作品を貫く共通の世界観設定。「キャラクターは時間を生きる」がコアルール。実体は10_Characters/FLY77STAR_Universe.md。VaultのFLY77STAR_Philosophy.mdとは別物なので注意）

Studio Philosophy
（「作品ではなく、人生を描く」というFLY77STARの創作面のアイデンティティ。Vault運用の考え方であるFLY77STAR Philosophyとは別物）

## ブランド・作品に関する用語

FLY77STAR.
（会社/レーベルの正式表記。読み:「フライスター」— 77は読まない。末尾のドットまで含めて正式。Luka Dončićの背番号77へのオマージュ）

SE77NTH.
（3人組AIガールズグループ。読み:「セブンス」— 77は読まない。詳細: [[10_Characters/SE77NTH|SE77NTH.]]）

Soul ID
（Higgsfield Soul 2.0でキャラクターの顔を固定登録する仕組み。同一キャラは全生成で同じSoul IDを使い、顔ドリフトを防ぐ。2026-07-11の制作方針変更で、Character Master(通常制作)には使用しないことに変更。アーティスト写真・ファッションカット・SNSビジュアル・雰囲気重視の一枚絵・コンセプトアートに限定使用。詳細: GitHub `IMAGE_GENERATION_POLICY.md`)

77ルール
（ブランド・グループ名の綴り中に「77」を視覚的に埋め込み、発音はしない命名規則。「なんて読むの?」という検索フックを作る）

era（時代管理）
（同一キャラクターのビジュアル大変更を「時代」として区別し、Soul IDも時代ごとに別登録する運用。例: KAIのMA-1期/US期）

## 制作に関する用語

Production Bible
（映像制作マニュアル。世界観・キャラクター・品質基準をまとめた文書）

CUT
（作品を構成する一つの映像カットの単位。CUT01〜のように番号で管理する。総数は作品ごとに異なる（例: WAKE UPは12、STAR LiGHTは50）。AI社員の [[04_Team/AI_Employees/CUT (AI Employee)|CUT (AI Employee)]] とは別物なので注意）

Panel Storyboard
（1つのCUTを、ショットサイズ・カメラワーク・構図・BPM同期まで書き込んだ複数パネルへ分解する映画レベルの絵コンテ形式。2026-07-10、WAKE UP CUT01で試作し正式採用(Studio OS v1.1)。実体は `Storyboard/CUT<番号>_絵コンテ.md`、テンプレートは `99_Templates/Panel_Storyboard_V1.md`）

Director Notes
（Panel Storyboardを補完する、監督の演出意図を記録する資料。観客に感じてほしいこと・演技指示・表情・視線・呼吸・間・カメラマンへの補足・照明/色の演出意図・編集時の注意・次パネルへの感情のバトンを記載する。全Panelには作らず、重要Panel(主人公初登場/感情変化/世界観説明/伏線/象徴カット/ラストカット)のみ作成する。2026-07-10、WAKE UP CUT01 Panel03で試作し正式採用(Studio OS v1.2)。実体は `Storyboard/CUT<番号>_Panel<番号>_DirectorNotes.md`、テンプレートは `99_Templates/Director_Notes_V1.md`。Panel Storyboard本体は変更しない）

EDIT_PLAN.md
（CUT_LIST.jsonから自動生成される、CUTごとの尺と編集指示をまとめた表）

edit_project.json
（Palmierに渡す、タイムライン全体の編集用データファイル）

CUT_LIST.json
（各CUTの役割・尺・セリフ有無・素材パスを管理する元データ）
