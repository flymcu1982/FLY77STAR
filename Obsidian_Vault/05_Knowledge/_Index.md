# Knowledge
（正式採用された会社ルール・ノウハウ）

Discovery Logに書かれた気づきのうち、繰り返し使えると確認されたものを昇格させる場所です。このVaultが「会社の脳」として育っていく、いちばんの成長エンジンです。

## 昇格の流れ

```
Discovery Log(今日の気づき)
   ↓ 何度も使われる(目安: 異なる日に3回以上)
昇格候補 (#昇格候補 タグを付ける)
   ↓ 未レビューのDiscovery Logが10件たまったら Knowledge Review
Knowledge(繰り返し使える知識)
```

## 昇格の基準(目安)

- 同じ気づきが、**異なる日に3回以上**、実際に役立った(参照された/リンクされた)。
- あくまで目安。明らかに重要なら1回でもKnowledge Reviewを待たずに昇格させてよい。逆に3回使われても、状況次第(そのCUT・その日限りの話など)で昇格させない判断もあってよい。
- 迷ったら [[00_Start_Here/Vault運用ルール|Vault運用ルール]] の「原液と精製後を分ける」考え方に立ち返る。

## Knowledge Review(いつやるか)

- トリガーは**カレンダーではなく件数**: 未レビューのDiscovery Logエントリーが**10件たまったら**実施する。
- 「未レビュー」= `#昇格済み` も `#レビュー済み` も付いていないエントリーのこと。数えるときはDiscovery Logフォルダをこの2つのタグで検索し、どちらも無いものを数える。
- 件数トリガーにしているのは、閑散期に無理やり見直しをこなす必要をなくし、実際に気づきが溜まった時だけ手を動かすため([[FLY77STAR_Philosophy|Philosophy]] 参照)。まだ運用実績が浅いので、10件という数字自体も実データを見ながら調整してよい。

## やり方

1. Discovery Logの発見を再利用するときは、新しいノートから元のエントリーへ `[[リンク]]` を貼る。(Obsidianの「Linked mentions」で参照回数が自然に見える)
2. 3回以上使われたと気づいたら、そのエントリーに `#昇格候補` タグを追加する。
3. 未レビューのエントリーが10件たまったらKnowledge Reviewを行い、`#昇格候補` を中心に全件に目を通す。
4. 昇格が決まったら、該当サブフォルダに新規ノートを作る([[99_Templates/Knowledge_note|テンプレート]] を使用)。元のDiscovery Logエントリーは `#昇格候補` を `#昇格済み` に変え、末尾に一行 `→ 昇格: [[新しいKnowledgeノート]]` を追記する。
5. 昇格させなかったエントリーにも `#レビュー済み` を付ける。これで「未レビュー」件数がゼロに戻り、次の10件のカウントが始まる。

タグの付け替えと、この1行の追記だけは「Discovery Logは書き換えない」ルールの例外として認めます。発見の内容そのものは変えません。([[00_Start_Here/Vault運用ルール|Vault運用ルール]] 参照)

## Knowledgeノートの中身

- 元ネタ: どのDiscovery Logエントリーから来たか(リンク)
- 内容: 再利用できる知識そのもの(短く)
- 使える場面: いつ使うと良いか
- 最終確認日: いつ有効性を確認したか
- 状態: 有効 / 要見直し / 廃止

## 大事な注意: GitHubのルールとの関係

ここでの「正式採用」は、あくまでこのVault内で「繰り返し使えると確認された」という意味です。GitHub上の制作ルール(`OPERATING_MANUAL.md` 等)を上書きする権限は持ちません。

Knowledgeに書いた内容が、本当は今後すべての制作に適用すべき正式ルールだと分かったら、`03_Company/Decisions/` を通してYUの承認を得た上で、GitHub側の該当ファイルを直接更新してください。ここに書くだけでは正式ルールにはなりません。

## Knowledgeの定期レビュー(古くなるのを防ぐ)

Knowledgeも放っておくと古くなります。年次で全ノートの「最終確認日」を見直し、まだ有効なら日付を更新、古くなっていたら状態を「要見直し」に変え、使われなくなったものは [[08_Archive/_Index|Archive]] へ移してください。

- `Style_Guides/` — 色・カメラ・音などのノウハウ(補足解説。ルールそのものは`OPERATING_MANUAL.md`が正本)
