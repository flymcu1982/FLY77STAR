# WAKE_UP

## 改善点

- Scene2(CUT05〜08)制作時、CUT07『[[KAI]]初登場、コックとしてのみ登場』という演出方針を、既存の[[Panel Storyboard]]本体の記述(内部演出メモに『謎めいた登場感』等)と、新たに作成した[[Director Notes]]の抑制方針との間でニュアンスを調整する必要が生じた。今回はPanel Storyboard本体を変更せずDirector Notes側で方針を明記して対応したが、今後『演出トーンの後出し調整』が増える場合、Panel Storyboard作成時点で演出トーン(観客に見せる強度)を項目として持たせるかどうか、運用結果を見て検討したい(Studio OS自体は今回改修せず)。
- Scene3制作で[[MIU]]の違和感インデックス(1〜10、キャラクター固有の主観指標)を新設し、Scene2導入のReality Scale(1〜10、世界の客観指標)と併用した。2指標の役割分担(客観/主観)は今回明確に機能したが、指標が増えると[[Director Notes]]作成時の判断コストも増える。次回作以降でこの2指標をどこまで標準化するか、運用結果を見て検討したい(Studio OSの改修は今回行わず)。
- 画像生成エンジン方針変更(Higgsfield/Nano Banana→Higgsfield/GPT Image、Soul 2はCharacter Master不使用)に伴い、Reference/CHAR_*_[[MASTER]].md(5点)は新方針に合わせて更新したが、Storyboard/CUT<番号>.md・CUT<番号>_絵コンテ.md・[[Director Notes]]・Edit/[[EDIT_PLAN.md]]内に残る『[[Soul ID]] locked for consistency』等の表記は遡及的に修正していない(対象30ファイル以上、Phase1以前に完成・[[レビュー済み]]の成果物のため今回はスコープ外とした)。実運用上は支障ないと判断するが、表記の統一が必要な場合は別途まとまった修正作業として計画したい。

## 発見

- Higgsfield MCPツール疎通の切り分け結果(2026-07-11、技術検証タスク、Production非対象)。通ることを確認: show_medias、show_reference_elements(action=list)。通らないことを確認(すべて同一エラー): models_explore(action=search)、show_generations(type=video)、generate_video(params.get_cost=true、モデルkling3_0_turbo・prompt「A single red apple on a white table. Static shot.」・duration=5、参照画像なしの最小コスト構成での見積り取得も含む)。いずれも同じエラー文言「Streamable HTTP error: Error POSTing to endpoint: MCP tool call requires approval」で、HTTPステータスコードや追加詳細は取得できず。get_cost=trueによるコスト見積りプリフライト(実ジョブ投入なし)ですら同じ理由でブロックされたことから、このゲートは特定のツール区分(カタログ検索・履歴閲覧・生成系)に対するサーバー側/アカウント側の制限である可能性が高いと推測(未確定)。ローカルの[[Claude Code]]権限設定(.claude/settings.json等)は本プロジェクト・ユーザーいずれのレベルにも存在せず、~/.claude.jsonのmcpServersも空{}であることを確認済み — 「ローカルallowlist未登録が原因」という仮説とは矛盾する。加えて、本セッションはmanaged remote execution environmentであり、Higgsfield MCPサーバーは環境側から提供されている(ローカルmcpServers設定ではない)。テスト直前にMCPサーバーが一度切断・再接続する挙動も観測(こちらから操作したものではない)。今後の運用改善のため記録。
